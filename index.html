O que é Programação Orientada a Objetos?

Se você nunca ouviu falar de POO, comece aqui! Vamos construir a base juntos usando analogias simples do dia a dia.

A Grande Mudança de Paradigma

Na Programação Estruturada/Procedural (como em C ou Python básico), escrevemos código como uma receita de bolo: uma sequência de instruções passo a passo de cima para baixo.

Na Programação Orientada a Objetos (POO), organizamos o programa em torno de "entidades" chamadas Objetos. Em vez de funções soltas, criamos módulos que possuem seus próprios dados (atributos) e ações (métodos).

A Metáfora do Smartphone

Pense no seu Smartphone como um Objeto:

  • Características (Atributos): Modelo, Cor, Nível de Bateria, Ligado (true/false).
  • Ações (Métodos): ligarTela(), fazerChamada(String numero), enviarMensagem().

O smartphone é um objeto real. Mas antes dele existir, a fábrica tinha um desenho técnico / molde. Esse molde é a Classe. A classe define o modelo; o smartphone no seu bolso é o Objeto (instância).

Os 4 Pilares da POO

  1. Encapsulamento — Esconder dados internos e expor apenas o necessário (como um controle remoto: você aperta botões sem saber a eletrônica interna).
  2. Herança — Criar classes a partir de outras, reaproveitando código (como um Smartphone que herda tudo de Dispositivo Eletrônico).
  3. Polimorfismo — Um mesmo método se comporta de formas diferentes dependendo do objeto (como tocar(): um Violão toca de um jeito, um Piano de outro).
  4. Abstração — Focar no que é essencial, ignorando detalhes irrelevantes para o contexto.

Por que usamos POO?

  1. Organização — Cada objeto cuida de suas próprias informações.
  2. Reutilização — Se você já tem o molde "Celular", pode criar 1000 celulares diferentes.
  3. Segurança — Proteger dados sensíveis com encapsulamento (private + getters/setters).
  4. Manutenção — Alterar um módulo sem quebrar todo o restante do sistema.
Para a prova: Os 9 conceitos cobrados neste guia são os blocos fundamentais da POO em Java. Domine cada um na teoria E na prática!

Estruturada vs Orientada a Objetos

Compare os dois paradigmas:

Programação Estruturada:
Foco em Como fazer — sequência de funções que processam dados soltos.
Orientação a Objetos:
Foco em Quem faz — objetos com responsabilidades que interagem entre si.
"Classes moldam o mundo; Objetos dão vida ao código."

Classe

O blueprint ou molde que define a estrutura de objetos.
